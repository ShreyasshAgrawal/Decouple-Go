How To Use This Guide

This guide is for someone new to Go and new to this project. It explains the code line‑by‑line for the most important files. Read it in order. Keep the code open in your editor and follow along.

If you see a word you don’t know, search in the “Go Basics” section first.

Go Basics You Need

Packages and Files

  • Each folder is usually one package.
  • All .go files in a folder share the same package name at the top of the file.

Example: internal/archivedecouple/*.go are all in package archivedecouple.

Imports

At the top of each file you’ll see imports. They are like #include in C or import in Python/JS.

import (
  "fmt"
  "os"
  "decouple/internal/archivedecouple"
)

Functions and Methods

A function:

func add(a int, b int) int { return a + b }

A method is a function attached to a type:

func (h *imgHandler) Detect(...) bool { ... }

Here, h is the receiver (like this in other languages).

Structs

Structs are simple objects with fields.

type Report struct {
  Artifact Artifact
  Nodes    []Node
  Stats    Stats
}

Interfaces

Interfaces are contracts. If a type implements all methods, it satisfies the interface.

This project uses the Handler interface to plug in different format scanners.

Errors

Go uses explicit errors:

rep, err := handler.Decouple(...)
if err != nil { return nil, err }

Context

context.Context is how Go cancels work and sets timeouts. Look for ctx.

Goroutines and Channels

go func() { ... }() starts a goroutine (lightweight thread).

Channels are used to limit parallel work.

Defer

defer runs cleanup code at the end of the function.

Line‑By‑Line Walkthrough

1) CLI Entrypoint

File: cmd/decouple/main.go

Imports

  • context: cancellation and timeouts.
  • encoding/json: to encode output as JSON.
  • flag: command line arguments.
  • fmt, io, os: standard input/output utilities.
  • decouple/internal/archivedecouple: core scanner.
  • decouple/internal/artifact: determine file kind.
  • decouple/internal/report: output schema.

main()

  1. flag.StringVar(&inputPath, ...) and flag.StringVar(&outputPath, ...) define flags.
  2. flag.Parse() reads --input and --output from the command line.
  3. If inputPath is empty, print error and exit.
  4. kind := artifact.KindFromPath(inputPath) maps extension to a logical kind.
  5. archivedecouple.DecouplePath(...) runs the scanner.
  6. If output path is set, create file and write JSON there; otherwise write to stdout.

2) API Entrypoint

File: cmd/api/main.go

analyzeHandler

  1. pickUploadedFile(...) tries multiple field names (file, upload, artifact).
  2. Log request info with slog.
  3. Save upload to a temp file.
  4. Determine kind by filename extension.
  5. Create a context with timeout.
  6. Call archivedecouple.DecouplePath(...).
  7. Return the JSON report or an error.

pickUploadedFile

  • Reads multipart form data.
  • If no file found, returns a helpful error listing what fields were provided.

3) Core Scanner Orchestration

File: internal/archivedecouple/archivedecouple.go

Key types

  • runtimeState: tracks limits, temp dir, and semaphore for parallel work.
  • state.cleanup(): deletes the temp dir after scan.

DecouplePath

Line by line:

  1. RegisterDefaults() sets up the handler registry once.
  2. cfg = ensureConfig(cfg) fills defaults.
  3. Create state with a semaphore sem sized to MaxConcurrentScans.
  4. defer state.cleanup() ensures temp dir is removed.
  5. If displayName is blank, use filepath.Base(path).
  6. Log scan start.
  7. Call decoupleArchive(...) to do the real work.
  8. Fill stats and return report.

decoupleArchive

  1. Check ctx.Done() (cancellation).
  2. Open file and read first 512 bytes.
  3. FindHandler(header, displayName) chooses a handler.
  4. If unknown format, return error.
  5. If kind empty, map from handler’s format.
  6. Call handler.Decouple to scan top‑level.
  7. Prefix node paths with pathPrefix.
  8. If at max depth, stop.
  9. Build nodeIndex map for quick updates.
  10. Call handler.WalkNested to find nested archives.
  11. For each nested archive:
    • Extract to temp file via prepareNestedArchive.
    • Spawn goroutine to scan nested archive.
    • Merge results into parent report.

prepareNestedArchive

  1. Enforces depth and size limits.
  2. Tracks nestedArchivesScanned and temp disk usage.
  3. Creates temp file in scan’s temp dir.
  4. Copies nested content into that temp file.
  5. Returns temp path or sets error on node.

4) Handler Registry

File: internal/archivedecouple/registry.go

  • Handler interface defines Detect, Decouple, WalkNested.
  • RegisterDefaults order matters:
    • tar → zip → pe → img → fat → gzip.
    • First match wins.

5) Output Schema

File: internal/report/schema.go

Report

  • Artifact: info about the input.
  • Nodes: list of files/directories found.
  • Stats: totals and counters.

Node

Important fields:

  • Path, Type, SizeBytes
  • SHA256, HashError
  • CompressedSizeBytes for zip entries
  • PartitionIndex, StartOffsetBytes, EndOffsetBytes for .img partitions
  • FilesystemType for .img
  • Status for .img parse status

6) Format Detection

File: internal/detect/detect.go

DetectBytes checks magic bytes in this order:

  1. Gzip (1f 8b)
  2. Zip (PK)
  3. Tar (ustar at offset 257)
  4. PE (MZ)
  5. IMG (MBR 0x55AA or GPT EFI PART)
  6. Raw filesystem signatures (ext4, FAT32) treated as IMG

7) Safety Limits

File: internal/safety/limits.go

These protect against zip bombs and huge scans. Examples:

  • Max files: 1,000,000
  • Max total bytes: 32 GiB
  • Max nested archives: 1,000
  • Max depth: 3

Config overrides are in internal/scanconfig/config.go.

8) ZIP Handler

File: internal/archivedecouple/zip.go

  • Detects ZIP by magic bytes.
  • Reads all entries and normalizes paths.
  • Hashes files in parallel.
  • WalkNested peeks headers to find nested archives.

9) TAR Handler

File: internal/archivedecouple/tar.go

  • Supports plain tar and tar.gz.
  • Streams entries without loading entire file.
  • Enforces max files and max total bytes.
  • Hashes regular files.
  • WalkNested peeks into each entry to detect nested formats.

10) GZIP Handler

File: internal/archivedecouple/gzip.go

  • Treats gzip as a single file payload.
  • Uses embedded name if present.
  • Hashes payload up to limit.
  • If payload looks like archive, it can be nested scanned.

11) PE Handler

File: internal/archivedecouple/pe.go

  • Uses Go’s debug/pe.
  • Emits one node per section.
  • Detects overlay data after last section.
  • Allows nested scanning inside sections or overlay.

12) IMG Handler (Deep)

File: internal/archivedecouple/img.go

This file is the most complex. Read it slowly.

Detect

  • If extension is .img, return true.
  • If MBR magic 0x55AA, return true.
  • If GPT signature EFI PART, return true.

Signature probe

imgSignatureProbe reads bytes at known offsets to identify containers and filesystems.

It can detect:

  • Gzip, QCOW2, VMDK, VHDX containers (unsupported).
  • LVM, LUKS, NTFS, XFS, Btrfs signatures.

If it detects unsupported types, it sets analysis_status so the report explains why it’s unreadable.

Decouple flow

  1. Create report.Report.
  2. Create imgSignatureProbe to detect container status.
  3. diskfs.Open(path) opens the image.
  4. GetPartitionTable() tries to find partitions.
  5. If no partition table:
    • Treat whole image as a filesystem.
    • If parse fails, emit one node with hash_error and analysis_status.
  6. If partitions:
    • For each partition, decoupleFilesystem.
    • On failure, emit partition node with error.

openFilesystem

  1. Try d.GetFilesystem(partIndex).
  2. If that fails and signature says ext4, try ext4.Read.

decoupleFilesystem

  • Walk filesystem recursively.
  • Create a node for each file or directory.
  • Hash files in parallel (limit: MaxIMGConcurrentScans).
  • Detect OS hints:
    • /etc/os-release, /usr/lib/os-release, or Windows registry hive.

WalkNested

  • Even inside .img, it tries to detect nested archives (zip, tar, pe, etc.).
  • It can also detect archives directly in raw partition data.

IMG fallback behavior

If filesystem parse fails:

  • Add one node for the partition or root.
  • Fill filesystem_type if detected.
  • Set hash_error with parse error.
  • Add analysis_status explaining why it is unreadable.

Study Path (Detailed)

  1. Read cmd/decouple/main.go and follow the flags and JSON output.
  2. Read cmd/api/main.go and the HTTP flow.
  3. Read internal/archivedecouple/archivedecouple.go carefully.
  4. Read internal/archivedecouple/registry.go and memorize handler order.
  5. Read internal/detect/detect.go and internal/artifact/artifact.go.
  6. Read internal/report/schema.go.
  7. Read handlers: zip.go, tar.go, gzip.go, pe.go, img.go.
  8. Read internal/safety/limits.go and internal/safety/path.go.
  9. Read tests in internal/archivedecouple/*_test.go.
